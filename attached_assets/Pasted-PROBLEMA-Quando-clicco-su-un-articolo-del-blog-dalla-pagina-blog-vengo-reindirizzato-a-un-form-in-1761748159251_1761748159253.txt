PROBLEMA: Quando clicco su un articolo del blog dalla pagina /blog, vengo reindirizzato a un form invece di vedere l'articolo pubblicato.

OBIETTIVO: Creare una pagina pubblica per visualizzare i singoli articoli del blog.

ISTRUZIONI STEP-BY-STEP:

1. CREA ROTTA PUBBLICA PER BLOG POST
   - Crea una nuova pagina in `client/src/pages/BlogPost.tsx`
   - La rotta deve essere `/blog/:slug` dove slug è l'URL-friendly identifier dell'articolo
   - Questa pagina deve essere PUBBLICA (non protetta da autenticazione)

2. STRUTTURA DELLA PAGINA BLOG POST
   Seguendo le design_guidelines.md, la pagina deve includere:
   - Breadcrumb: Home > Blog > Titolo Articolo
   - Hero section con:
     * Immagine cover dell'articolo (aspect ratio 16:9)
     * Categoria tag (warm sand background)
     * Titolo articolo (h1, font Playfair Display)
     * Sottotitolo/excerpt
     * Metadati: data pubblicazione, tempo di lettura, autore
   - Contenuto articolo:
     * Renderizza il contenuto Markdown in HTML
     * Usa prose prose-lg per la tipografia
     * Mantieni max-w-4xl per la leggibilità
   - Sezione "Articoli Correlati" alla fine (3 articoli della stessa categoria)
   - CTA Newsletter (sfondo warm sand)

3. FETCHING DATI
   - Crea endpoint GET `/api/blog/:slug` nel backend
   - L'endpoint deve ritornare l'articolo con status "published"
   - Includi tutti i campi: title, subtitle, slug, coverImage, content, tags, category, author, publishedAt, readingTime
   - Se l'articolo non esiste o non è pubblicato, ritorna 404

4. AGGIORNA LA PAGINA /BLOG
   - Modifica le card degli articoli in `client/src/pages/Blog.tsx`
   - Ogni card deve linkare a `/blog/:slug` invece del form
   - Usa il componente Link di wouter: `<Link href={`/blog/${article.slug}`}>`
   - Mantieni lo stile attuale delle card

5. RENDERING MARKDOWN
   - Usa le librerie già presenti: remark, remark-parse, remark-rehype, rehype-sanitize, rehype-stringify
   - Sanitizza l'HTML per sicurezza
   - Aggiungi classi Tailwind per stilizzare il contenuto markdown

6. META TAGS SEO
   - Usa il helper usePageMeta già presente in `client/src/lib/seo.ts`
   - Imposta meta title: `{article.title} | Immobiliare Maggiolini`
   - Meta description: primi 155 caratteri del subtitle o content
   - Open Graph tags per condivisione social
   - JSON-LD structured data per Article

7. GESTIONE ERRORI
   - Se l'articolo non esiste, mostra pagina 404 personalizzata
   - Se c'è errore di caricamento, mostra messaggio d'errore user-friendly
   - Aggiungi loading state durante il fetch

8. STILE E DESIGN
   - Usa palette mediterranea (terracotta, warm sand, sage green)
   - Font: Playfair Display per titoli, Inter per body
   - Spacing consistente: py-20 per sezioni
   - Responsive: mobile-first approach
   - Dark mode supportato

9. VERIFICA CHE IL ROUTING FUNZIONI
   - Assicurati che la rotta `/blog/:slug` sia aggiunta in `client/src/App.tsx`
   - Verifica che non ci siano conflitti con rotte admin
   - La rotta admin per modificare articoli dovrebbe essere `/admin/blog/:id/edit` (diversa da quella pubblica)

10. TESTING
    - Testa che cliccando su un articolo dalla pagina /blog si apra l'articolo completo
    - Verifica che il Markdown sia renderizzato correttamente
    - Controlla che immagini e link funzionino
    - Testa su mobile e desktop
    - Verifica dark mode
    - Controlla che articoli non pubblicati (status draft) non siano accessibili pubblicamente

NOTE IMPORTANTI:
- NON modificare le rotte admin esistenti
- La pagina /admin/blog deve rimanere protetta e mostrare la lista per modificare articoli
- Usa i componenti UI già presenti in `client/src/components/ui`
- Segui lo stile delle altre pagine pubbliche (Home, Proprietà, Chi Siamo)
